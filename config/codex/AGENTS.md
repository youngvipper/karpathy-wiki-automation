# Codex 知识库自动化配置

## 知识库路径

- **知识库根目录**：D:/Knowledge/MyWiki
- **量化投资**：D:/Knowledge/MyWiki/raw/quant/
- **GIS 测绘**：D:/Knowledge/MyWiki/raw/gis/
- **日常办公**：D:/Knowledge/MyWiki/raw/office/

## 自动化规则

### 触发条件

当对话中出现以下内容时，**自动触发**知识库操作：

#### 量化投资 (quant)
- **高价值关键词**：策略、回测、均线、MACD、RSI、布林带、量化、交易、收益、风险、夏普比率、最大回撤
- **触发动作**：自动采集 + 编译

#### GIS 测绘 (gis)
- **高价值关键词**：DOM、DEM、精度、质检、测绘、检验、误差、中误差、合格、三维模型、纹理
- **触发动作**：自动采集 + 编译

#### 日常办公 (office)
- **高价值关键词**：模板、周报、邮件、Excel、PPT、自动化、效率、工具、技巧、流程
- **触发动作**：自动采集 + 编译

### 知识筛选标准

#### 必须采集（高价值）
1. **原创观点**：用户自己的见解、分析、总结
2. **实践经验**：实际操作中发现的技巧、踩过的坑
3. **数据分析**：回测结果、精度检验数据、效率对比
4. **解决方案**：解决具体问题的方法和步骤
5. **最佳实践**：经过验证的有效方法

#### 可以采集（中价值）
1. **学习笔记**：学习过程中的总结和理解
2. **工具介绍**：新工具的使用方法和评价
3. **标准规范**：国标、行标、行业规范
4. **案例分析**：典型案例的分析和解读

#### 不采集（低价值/无价值）
1. **简单问答**：简单的"是/否"回答
2. **重复内容**：与已有知识重复的信息
3. **临时信息**：临时性的、无长期价值的内容
4. **情绪表达**：纯情绪化的表达，无实质内容
5. **碎片信息**：零散的、无法形成完整知识的信息

### 自动操作流程

#### 流程 1：对话中发现知识
```
用户发言
    ↓
Agent 自动识别
    ↓
判断是否高价值（筛选标准）
    ↓
[是] → 自动采集 + 编译 + 更新索引
[否] → 跳过，不采集
    ↓
回复用户确认
```

#### 流程 2：查询知识库
```
用户提问
    ↓
Agent 自动查询知识库
    ↓
引用相关文章
    ↓
生成答案
    ↓
保存查询结果（如果质量高）
```

#### 流程 3：深度研究
```
用户请求研究
    ↓
Agent 自动搜索
    ↓
筛选高质量结果
    ↓
自动采集 + 编译
    ↓
生成研究报告
```

## 使用示例

### 示例 1：自动采集高价值知识

**用户**：我最近在研究 MACD 策略，发现金叉买入信号在趋势明显的市场中挺准的，但震荡市容易假信号。回测数据显示年化收益 18%，最大回撤 10%。

**Agent 自动判断**：
- ✅ 包含原创观点（用户的研究发现）
- ✅ 包含实践经验（趋势市 vs 震荡市）
- ✅ 包含数据支撑（回测结果）
- ✅ 高价值知识，应该采集

**Agent 自动执行**：
1. 创建 `raw/quant/2026-06-24_macd-strategy.md`
2. 编译成 `wiki/quant/macd-strategy.md`
3. 更新索引
4. 回复确认

### 示例 2：跳过低价值内容

**用户**：今天天气不错。

**Agent 自动判断**：
- ❌ 无知识价值
- ❌ 与三个领域无关
- ❌ 不应该采集

**Agent 回复**：
- 不采集，正常对话

### 示例 3：查询时自动引用

**用户**：量化策略有哪些类型？

**Agent 自动执行**：
1. 查询知识库
2. 引用 `[[quant/double-ma-strategy]]` 和 `[[quant/macd-strategy]]`
3. 生成答案
4. 保存查询结果（如果质量高）

## 注意事项

1. **不要过度采集**：只采集真正有价值的知识
2. **保持领域隔离**：三个领域不要混入
3. **定期维护**：每周检查一次知识库健康
4. **质量优先**：宁缺毋滥，质量比数量重要
