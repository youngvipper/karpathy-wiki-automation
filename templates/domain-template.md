# 领域定义模板

> 使用此模板定义你的知识领域。

## 领域定义格式

每个领域包含以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 领域的简短名称 | 技术研究 |
| 目录 | 领域的目录名 | tech |
| 子领域 | 该领域包含的子主题 | 编程语言、框架、架构 |
| 关键词 | 触发自动采集的关键词 | 代码、架构、设计模式 |
| 目标 | 该领域要解决的问题 | 系统化管理技术知识 |
| 产出 | 该领域产出什么 | 技术文档、最佳实践 |

## 如何定义你的领域

### 步骤 1：确定主要领域

问自己：
- 我主要在哪些方面积累知识？
- 我的知识主要来自哪些场景？
- 我希望知识库帮我解决什么问题？

### 步骤 2：定义子领域

对于每个主要领域：
- 该领域包含哪些子主题？
- 这些子主题之间有什么关系？
- 如何组织这些子主题？

### 步骤 3：确定关键词

对于每个领域：
- 哪些关键词能触发自动采集？
- 这些关键词是否足够独特？
- 是否会与其他领域混淆？

### 步骤 4：明确目标和产出

对于每个领域：
- 该领域要解决什么问题？
- 该领域产出什么？
- 如何衡量该领域的价值？

## 领域隔离原则

### 目录隔离

每个领域独立目录：
```
raw/
├── <domain1>/
├── <domain2>/
└── <domain3>/

wiki/
├── <domain1>/
├── <domain2>/
└── <domain3>/
```

### 标签隔离

每篇文章必须标注 `domain` 字段：
```yaml
---
title: 文章标题
domain: <domain>
---
```

### 查询隔离

查询时可按领域过滤：
```bash
/karpathy-wiki query --domain <domain> <问题>
```

## 常见领域组合

### 组合 1：技术 + 管理 + 学习

适合：技术管理者、全栈工程师

- 技术研究：代码、架构、技术选型
- 项目管理：需求、排期、团队协作
- 学习笔记：读书、课程、技能提升

### 组合 2：研究 + 实践 + 创作

适合：研究者、内容创作者

- 学术研究：论文、理论、方法论
- 实践探索：实验、数据分析、案例
- 内容创作：文章、视频、演讲

### 组合 3：专业 + 兴趣 + 生活

适合：有明确专业方向的人

- 专业知识：行业知识、技能、标准
- 兴趣爱好：个人兴趣、业余项目
- 生活管理：健康、财务、时间管理
