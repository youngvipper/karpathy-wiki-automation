# 知识筛选标准

## 核心原则

**宁缺毋滥，质量比数量重要**

## 筛选维度

### 维度 1：价值判断

#### 高价值（必须采集）

1. **原创观点**
   - 用户自己的见解和分析
   - 独特的思考角度
   - 创新的解决方案

2. **实践经验**
   - 实际操作中发现的技巧
   - 踩过的坑和解决方案
   - 验证过的方法

3. **数据分析**
   - 统计数据和实验结果
   - 对比分析
   - 趋势分析

4. **解决方案**
   - 解决具体问题的方法
   - 完整的操作步骤
   - 可复用的模板

5. **最佳实践**
   - 经过验证的有效方法
   - 行业认可的标准做法
   - 优化后的流程

#### 中价值（可以采集）

1. **学习笔记**
   - 学习过程中的总结
   - 对概念的理解
   - 知识点的梳理

2. **工具介绍**
   - 新工具的使用方法
   - 工具的优缺点评价
   - 适用场景分析

3. **标准规范**
   - 国标、行标、行业规范
   - 技术文档
   - 操作手册

4. **案例分析**
   - 典型案例的分析
   - 成功/失败的经验
   - 可借鉴的方法

#### 低价值（不采集）

1. **简单问答**
   - "是/否"回答
   - 简单的事实确认
   - 无深度的讨论

2. **重复内容**
   - 与已有知识重复
   - 换汤不换药的信息
   - 无新意的表达

3. **临时信息**
   - 临时性的、无长期价值
   - 一次性的信息
   - 过时的内容

4. **情绪表达**
   - 纯情绪化的表达
   - 无实质内容
   - 无法形成知识

5. **碎片信息**
   - 零散的、无法形成完整知识
   - 缺乏上下文
   - 无法复用

### 维度 2：领域匹配

#### 领域判断

- 内容是否属于已定义的领域？
- 内容是否与多个领域相关？
- 内容应该归入哪个领域？

#### 关键词匹配

- 内容是否包含领域关键词？
- 关键词是否足够独特？
- 是否需要调整关键词？

### 维度 3：完整性检查

#### 信息完整性

- ✅ 有明确的主题
- ✅ 有完整的描述
- ✅ 有具体的例子
- ✅ 有可操作的步骤

#### 信息缺失

- ❌ 主题不明确
- ❌ 描述不完整
- ❌ 缺乏例子
- ❌ 无法操作

## 筛选流程

### 步骤 1：初步判断

1. 识别对话中的知识内容
2. 判断是否属于已定义的领域
3. 判断是否有采集价值

### 步骤 2：价值评估

1. 根据价值判断标准评估
2. 确定价值等级（高/中/低）
3. 决定是否采集

### 步骤 3：完整性检查

1. 检查信息是否完整
2. 检查是否有缺失
3. 决定是否需要补充

### 步骤 4：执行操作

1. **高价值**：立即采集 + 编译
2. **中价值**：标记待采集，等用户确认
3. **低价值**：跳过，不采集

## 实际案例

### 案例 1：高价值知识

**用户发言**：
```
我最近在研究微服务架构，发现对于小团队来说，
单体架构可能更合适。因为微服务带来的运维复杂度
远超其收益，尤其是在团队少于 10 人的情况下。
```

**筛选判断**：
- ✅ 原创观点：用户的研究发现
- ✅ 实践经验：小团队的适用性
- ✅ 领域匹配：技术研究
- ✅ 信息完整：有观点、有理由、有结论

**操作**：立即采集 + 编译

### 案例 2：中价值知识

**用户发言**：
```
我今天学习了 Docker 的基本用法，主要内容包括...
```

**筛选判断**：
- ✅ 学习笔记：用户的学习总结
- ✅ 领域匹配：技术研究
- ✅ 信息完整：有主题、有内容
- ⚠️ 价值中等：是学习笔记，非原创观点

**操作**：标记待采集，等用户确认

### 案例 3：低价值内容

**用户发言**：
```
今天天气不错，心情很好。
```

**筛选判断**：
- ❌ 无知识价值：纯情绪表达
- ❌ 领域不匹配：与任何领域无关
- ❌ 信息不完整：无法形成知识

**操作**：跳过，不采集

## 自动化实现

### Agent 自动识别

Agent 在对话中自动：
1. **关键词识别**：识别领域关键词
2. **上下文分析**：分析对话上下文
3. **价值判断**：根据标准判断价值
4. **自动执行**：高价值自动采集

### 用户触发

用户通过自然语言触发：
- "把这个收录到知识库"
- "记录这个发现"
- "保存这个方案"

Agent 自动：
1. 理解用户意图
2. 判断知识价值
3. 执行采集操作

## 质量控制

### 采集时

1. **价值判断**：根据标准判断
2. **领域匹配**：判断所属领域
3. **完整性检查**：确保信息完整

### 编译时

1. **去重处理**：避免重复内容
2. **结构化整理**：统一格式
3. **关联建立**：建立知识关联

### 维护时

1. **质量审查**：定期检查质量
2. **过时清理**：删除过时内容
3. **索引更新**：保持索引最新
