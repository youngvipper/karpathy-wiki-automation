# 知识库目标

构建个人知识管理系统，覆盖三个独立领域，实现知识的结构化沉淀和复利增长。

## 核心问题

1. 如何系统化管理量化投资的研究成果和交易策略？
2. 如何沉淀 GIS 测绘质检的经验和自动化脚本？
3. 如何积累日常办公的效率工具和最佳实践？
4. 如何让知识在三个领域内分别生长，互不污染？
5. 如何让 AI Agent 自动维护和更新知识库？

## 研究范围

### 领域一：量化投资
- **子领域：** 策略研究、技术指标、回测验证、市场洞察
- **时间跨度：** 持续积累
- **深度：** 策略逻辑 + 回测数据 + 实盘验证
- **产出：** 文章、视频、交易记录

### 领域二：GIS 测绘质检
- **子领域：** DOM 精度检验、DOM 影像质量、三维模型质检、可视化
- **时间跨度：** 持续积累
- **深度：** 检验标准 + 自动化脚本 + 案例分析
- **产出：** 自动化工具、质检报告、技术文档

### 领域三：日常办公
- **子领域：** 模板库、自动化流程、效率工具、办公技巧
- **时间跨度：** 持续积累
- **深度：** 实用技巧 + 可复用模板 + 自动化脚本
- **产出：** 模板、脚本、最佳实践

## 领域隔离规则

**严格隔离**：三个领域的知识不得互相混入
- 量化投资内容 → `wiki/quant/`
- GIS 测绘内容 → `wiki/gis/`
- 日常办公内容 → `wiki/office/`

**标签规范**：
- 每篇文章必须标注 `domain` 字段
- 查询时使用 `--domain` 参数过滤

## 研究策略

### 量化投资
- **策略追踪：** 优先 Ingest 经典策略和最新研究
- **回测验证：** 每个策略必须有回测数据支撑
- **实盘记录：** 记录实盘表现和经验教训
- **文章创作：** 将研究成果转化为文章和视频

### GIS 测绘质检
- **标准追踪：** 优先 Ingest 国标、行标和行业规范
- **案例积累：** 记录典型质检案例和解决方案
- **脚本开发：** 持续开发和优化自动化脚本
- **可视化探索：** 探索新的可视化技术和方法

### 日常办公
- **模板积累：** 收集和整理常用模板
- **技巧分享：** 记录实用的办公技巧
- **自动化开发：** 开发提升效率的自动化脚本

## 质量标准

### 所有领域通用
- 观点必须有来源支撑
- 脚本必须包含使用说明
- 模板必须包含使用场景

### 量化投资专用
- 策略必须包含入场/出场条件
- 策略必须包含止损止盈规则
- 策略必须包含回测结果

### GIS 测绘质检专用
- 检验方法必须引用相关标准
- 脚本必须包含输入输出说明
- 案例必须包含问题描述和解决方案
